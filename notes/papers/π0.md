# π0: A Vision-Language-Action Flow Model for General Robot Control

- Year / Venue：2024 arXiv，当前 PDF 为 2026 v 4
- Lab / Org：Physical Intelligence
- Type：VLA / robot foundation model / flow matching policy / dexterous manipulation
- Open-source（code, dataset）：论文未说明完整开源代码、权重或训练数据；使用了<mark style="background: #FF5582A6;">部分开源数据如 OXE、Bridge v 2、DROID</mark>，但核心 π 数据集为自有数据
- Link：
    - Blog：[https://physicalintelligence.company/blog/pi0](https://physicalintelligence.company/blog/pi0)
    - Arxiv：[https://arxiv.org/abs/2410.24164](https://arxiv.org/abs/2410.24164)

> 一句话定位：
> π0 是一个面向通用机器人控制的 VLA 模型，用预训练 VLM 继承语义理解能力，再通过 action expert + flow matching 生成连续 action chunk，从而支持高频、灵巧、多机器人本体的真实机器人控制。
>
> Key words：
> VLA, robot foundation model, flow matching, action expert, action chunking, cross-embodiment training, pre-training / post-training, dexterous manipulation

## Problem

这篇论文主要解决机器人基础模型中的三个瓶颈：

1. **Data**：机器人数据远少于互联网图文数据，单任务/单机器人数据难以覆盖真实世界变化。
2. **Generalization**：机器人策略需要跨任务、跨环境、跨物体、跨机器人本体泛化。
3. **Dexterous control**：传统 VLA 常把动作离散化为 token，自回归生成动作，但这对高频连续控制、灵巧操作、长时程任务不够合适。

论文试图回答的问题是：
**如何设计一个既能继承 VLM 语义知识，又能输出高精度连续动作，并能通过大规模跨本体数据训练的通用机器人策略模型？**

## Core idea

π0 的核心思路是：不要让 VLM 直接像 RT-2 / OpenVLA 那样输出离散 action token，而是在 VLM backbone 上加入一个专门处理机器人状态和动作的 **action expert**，并用 **flow matching** 生成连续动作分布。

模型先在大规模、多机器人、多任务数据上预训练，获得通用物理行为和恢复能力；再在高质量任务数据上 post-training，使模型能够更稳定、流畅地完成复杂灵巧任务。

## Contributions

- 提出一个新的 **flow matching VLA** 架构：在预训练 VLM 上加入 action expert，用连续动作建模替代离散动作 token。
- 支持 **action chunking**：一次生成未来多个连续动作，用于高频控制和灵巧任务。
- 使用 **cross-embodiment training**：在单臂、双臂、移动操作机器人等多种 embodiment 上联合训练。
- 提出类似 LLM 的机器人训练范式：**pre-training + post-training**。
- 在真实机器人任务上展示较强能力，包括 laundry folding、table bussing、assembling box、packing eggs、to-go box 等复杂任务。
- 实验表明：预训练模型在复杂任务上比 scratch 或仅 out-of-box 设置更有效，尤其在困难任务中收益更明显。

## Method

### Inputs / outputs

- Observation：
    - 多路 RGB 图像，通常每个机器人有 2 或 3 个 camera view
- Language：
    - 自然语言任务指令，例如 fold the shirt / bus the table
- State / Proprioception：
    - 机器人本体状态，如 joint angles
- Action：
    - 连续 action chunk
    - 默认 action horizon：H = 50
    - 不同机器人有不同 action space，通过 padding / masking 对齐到统一维度
- Other：
    - 可结合 high-level VLM policy，将长任务拆成中间语言指令

### Pipeline

$$
\text{多视角图像 + 语言指令 + proprioceptive state} \rightarrow \text{PaliGemma VLM backbone} \rightarrow \text{action expert} \rightarrow \text{flow matching 生成连续 action chunk} \rightarrow \text{robot controller 执行动作}
$$

更具体地说：

1. 图像和语言输入进入 VLM backbone。
2. 机器人状态和 noisy action chunk 进入 action expert。
3. Transformer 通过 attention 融合语义、视觉、本体状态和动作信息。
4. 模型学习从噪声动作恢复到真实动作的 vector field。
5. 推理时从随机噪声动作开始，通过 Euler integration 迭代生成最终 action chunk。

### Key design

- 最关键的结构设计：
    - **Action expert**：专门处理机器人状态和动作 token，与 VLM backbone 共享 attention 交互。
    - **Flow matching**：用连续生成模型输出动作，而不是把动作离散成 token。
    - **Action chunking**：一次生成未来 H 步动作，提升动作平滑性和高频控制能力。
    - **Pre-training / post-training recipe**：先用多样数据学通用能力，再用高质量任务数据学稳定执行。
- 和普通 VLA / policy / world model 的区别：
    - 和 RT-2 / OpenVLA 相比：π0 不主要依赖离散 action token，而是输出连续动作。
    - 和 Diffusion Policy 相比：π0 不是单纯任务级 imitation policy，而是接入 VLM backbone 的 robot foundation model。
    - 和 WAM / WorldVLA 相比：π0 不预测未来视频或世界状态，因此不是 world model；它主要是 action model / policy model。

## Evaluation

- Setting：
    - real robot
    - out-of-box evaluation
    - fine-tuning / post-training evaluation
    - high-level VLM policy + π0 low-level control
- Task：
    - seen / pretraining-related tasks
    - complex downstream tasks
    - long-horizon
    - dexterous manipulation
    - multi-stage tasks
    - language-conditioned tasks
- Dataset / Benchmark：
    - 自有 π dataset
    - OXE subset / OXE Magic Soup
    - Bridge v 2
    - DROID
    - 7 种机器人配置
    - 68 个任务
    - 约 10,000 小时机器人数据
- Baseline：
    - OpenVLA
    - Octo
    - π0-small，无 VLM 初始化的小模型
    - compute parity π0
    - scratch training
    - out-of-box π0
    - full pre-trained + fine-tuned π0
- Metric：
    - normalized task progress / task score
    - 每个任务多次 rollout
    - 1.0 表示完整成功，部分完成给部分分数
- Main result：
    - out-of-box 设置下，π0 在 shirt folding、bussing、grocery bagging、toast out of toaster 等任务上整体显著优于 OpenVLA、Octo、π0-small。
    - post-training 设置下，完整 pre-trained + fine-tuned π0 在 laundry folding、table bussing、assembling box、packing eggs、to-go box 等复杂任务上表现最好。
    - 预训练对困难任务帮助更明显，说明大规模多样数据不仅提升直接泛化，也提升后训练的起点质量。
- Convincing?
    - 比较有说服力，因为实验覆盖真实机器人、多 embodiment、灵巧操作和长时程任务，不只是仿真 benchmark。
    - 但也要注意：模型结构、数据规模、私有数据质量、训练步数和 post-training recipe 同时变化，不能把全部提升简单归因于 flow matching 架构本身。
    - OpenVLA 等 baseline 在这类高频 action chunk / dexterous setting 下天然不占优，因此对比更能说明 π0 适合灵巧控制，但不等价于全面证明 π0 在所有 VLA 任务上优于 OpenVLA。

## Limitations

- 复现门槛极高：依赖约 10,000 小时真实机器人数据、大量私有数据和多种真实机器人平台。
- 开源程度有限：论文没有说明完整开放模型权重、训练代码和核心数据集。
- 结构收益和数据收益难以完全分离：π0 的优势来自模型、数据规模、训练配方、机器人平台和后训练共同作用。
- 不是 world model：模型不显式预测未来视觉状态，对环境动力学的建模主要体现在 action policy 中。
- 对普通实验室或本科项目而言，不适合作为直接复现对象，更适合作为理解前沿 VLA 架构和训练范式的代表工作。

## Takeways

- 对我理解具身智能最有用的点：
    - 通用机器人策略不只是“VLM + action head”，动作表示本身非常关键。
    - 离散 action token 适合语言模型框架，但对高频、连续、灵巧控制存在天然限制。
    - Flow matching / diffusion-style action generation 是当前 VLA 走向 dexterous control 的重要路线。
    - 机器人 foundation model 也逐渐形成类似 LLM 的训练范式：pre-training 获得广泛能力，post-training 获得任务级高质量行为。
    - Cross-embodiment training 是通用机器人模型的核心问题之一，不同机器人 action/state/camera 维度不一致，需要统一建模和 masking / padding 机制。
- 对后续项目 / 复现 / 汇报可能有用的点：
    - 汇报时可将 π0 归类为：**continuous-action VLA / flow matching VLA / robot foundation model**。
    - 和 OpenVLA 对比时，重点说：
        - OpenVLA：离散 action token，开源友好，适合入门和复现。
        - π0：连续 action chunk，适合高频灵巧控制，但复现门槛极高。
    - 和 WorldVLA / WAM 对比时，重点说：
        - π0 是 action model，不预测 future video。
        - WAM / WorldVLA 试图把 action prediction 和 world prediction 联合起来。
    - 对暑研更现实的切入点：
        - 不复现 π0 全模型；
        - 可以理解其思想后，在 LeRobot / ACT / Diffusion Policy / OpenVLA 上做小规模 continuous action head、action chunking 或 post-training 实验。

## Question and future following

- 最需要弄清楚的问题：
    - Flow matching action expert 相比 Diffusion Policy 的本质优势在哪里？是 VLM backbone 带来的，还是 action expert 架构带来的？
    - 如果没有 10,000 小时机器人数据，小规模数据下 π0 类架构是否仍然有效？
    - π0 的 cross-embodiment padding / masking 机制在不同机器人 action space 差异很大时是否会损失动作语义？
    - π0 是否能学习真正未见过的新 physical skill，还是主要在已有技能空间内做组合和泛化？
    - Post-training 数据质量如何影响最终复杂任务表现？
- 后续要追的论文方向：
    - RT-2：离散 action token VLA 的经典起点。
    - OpenVLA：开源 VLA baseline，适合理解可复现训练流程。
    - π0.5：π0 后续开放世界泛化方向。
    - WorldVLA / UVA：action model + world model 的统一建模方向。
    - DreamZero / WAM：用视频扩散模型的 world prior 来增强机器人策略。
    - Diffusion Policy / ACT：理解 action chunking 和连续控制的基础方法。
